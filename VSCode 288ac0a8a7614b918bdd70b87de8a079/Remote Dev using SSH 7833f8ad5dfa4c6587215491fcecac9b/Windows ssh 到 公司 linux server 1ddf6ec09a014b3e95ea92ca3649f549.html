Windows ssh 到 公司 linux server

1. System requirements

  • 需要確認已安裝以下三個東西：
    • VSCode
    • Remote Development extension pack
    • OpenSSH compatible SSH client

1.1 VSCode

  • 這個早就裝好了，也不用截圖了

1.2 Remote Development extension pack

  • 就是 VSCode 上的 extension

1.3 OpenSSH compatible SSH client

  • 在 local 端，需要安裝 OpenSSH compatible SSH client
  • 可以看到，對 Windows 來說，有兩個選擇：
    • 安裝 Windows OpenSSH client ：但還要跟 mis 申請，很麻煩
    • 安裝 Git for Windows : 這個我之前用 git 時就裝過了，所以我等等要用這個。

2. 設定本地的 public key

2.1 確認一下有沒有生成過 id_rsa.pub

  • 用 git bash 打開 terminal
  • id_rsa.pub 預設會放在 /c/Users/19002329/.ssh 這個資料夾下，所以我們可以這樣檢查：
cd ~ # 切到使用者的目錄，對我來說，就是 /c/Users/19002329
cd .ssh # 切到此資料夾中
ls # 看一下裡面有沒有 id_rsa.pub

2.2 如果沒有，自己生成一個

cd .ssh # 如果連 .ssh 資料夾都沒有，就自己建立一個： mkdir .ssh
ssh-keygen

3. 把 public key 傳到遠端 linux server 上

  • 用winscp，先登入到 linux server，以 hadoop測試區當範例
  • 登進去後，進到 home/hadoop/.ssh 資料夾中，編輯 authorized_keys 這個檔案 (如果沒有這個檔案，幫他新增一個)
  • 內容就直接把筆電的 public key 的內容貼上 (在自己的筆電，用記事本，打開 /c/Users/19002329/.ssh/id_rsa.pub 這個檔案)

4. 回 VSCode 做設定

  • 在 VSCode 中，用 cmd+shift+p 開啟設定，並 key 入 Remote-SSH: Open SSH Configuration File ，並打開我目前目錄下的 ssh config 檔案 ( User/hanklee/.ssh/config )。如果沒有這個檔案，就自己建一個。
  • 在此文件中，做以下輸入：
Host hadoop-s1-testing
  HostName 172.16.99.233
  User hadoop
  IdentityFile C:\Users\19002329\.ssh\id_rs
  • 說明一下：
    • Host: [必填] 這邊就是自己幫遠端 server 命名，我寫 hadoop-s1-testing 的意思是，帳號用 hadoop，ip是 s1測試區 的意思
    • HostName: [必填] 就是 ip
    • User: [必填] 也是剛剛連線資訊中有提到的登入 user ，目前是 ubuntu
    • IdentityFile: [選填] 填入這台電腦 private key 擺放的位子
  • 如果最後一行的 identityFile 不寫的話，那也是可以連線，只是他會要你輸入密碼。而且是，你每瀏覽一個資料夾，他都要你輸入一次，非常麻煩
  • 那如果有設定 IdentityFile，那他會用 ssh 認證，就是去比對 遠端 server 的 .ssh 資料夾中 的 authorized_keys 這個檔案下，有沒有 public key 的內容，能和你這邊寫的筆電(local端) 的 private key 的內容對起來。如果對得上，就連線成功

5. 選定 git for windows 來幫忙做 ssh

  • 要做到 SSH 連線，需要有個 client 端。雖然 windows 有出 Windows OpenSSH client ，但還要跟 mis 申請，很麻煩。所以，我們直接指定用已經安裝過的 git for windows 來處理就好
  • 打 ctrl + shift + p 進入設定，打 Open settings (JSON) 進入 json 的設定檔，並把以下內容加入即可：
    "remote.SSH.path": "D:\\Programs\\Git\\usr\\bin\\ssh.exe"
  • 那個路徑，前面是你安裝 git for windows 時的路徑，我是裝在 D:\\Programs\\Git ，所以這樣寫。那如果是預設的話，應該會裝在 C:\Program Files\Git 這個下面，那路徑再改一下就好。
  • 設定完後，VSCode 就知道要用哪個 ssh client 去執行認證了。

5. 連線吧！！

  • 點 VSCode 左下角的綠色連線 ICON → 選 Connect to Host → 選你剛剛定義好的 Host (e.g. hadoop-s1-testing) → 連線成功！！
  • 現在只要 open file 或 open folder，都可以取得這個遠端的資料了。
