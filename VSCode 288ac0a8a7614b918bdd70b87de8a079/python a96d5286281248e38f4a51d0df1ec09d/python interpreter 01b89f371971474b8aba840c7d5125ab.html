python interpreter

1. 設定 VSCode 要用哪個 python 來跑code

1.1 先知道自己的電腦有哪些 python

  • 在自己的 terminal 裡面
    • call which python ，可以知道目前電腦用的 python 路徑在哪
    • call python --version ，可以知道這個 python的 version
  • 以公司的 windows 筆電來說
    • which python → /d/Programs/anaconda3/python
    • python --version → python 3.7.4
  • 以我自己的 mac 筆電來說，我是用 pyenv 來管理 python版本，所以指令比較多：
    • pyenv versions → 可以知道電腦中有哪些 python 版本。有加 * 的，表示目前正在用的版本
    • pyenv global 3.8.0 → 指定我要用 3.8.0 的版本。 global 的意思是說，我不只在現在的 terminal的session要用 3.8.0，我是要在所有(global)的session中，都用3.8.0
    • python --version → 這和 pyenv無關了，就是剛剛學過的系統指令，會得到現在在使用的 python版本。所以得到的結果是： python 3.8.0
    • which python → 可以得到目前這個 python 的路徑，結果是： /Users/hanklee/.pyenv/shims/python

1.2 讓 VSCode 吃到這個路徑下的 python

1.2.1 Windows的好簡單

  • 最簡單的方法： cmd+shift+p，打 python select interpreter ，然後用滑鼠選到你要使用的那個版本。以 windows公司筆電為例，我就會選到 Python 3.7.4 64-bit ('base': conda) D:\Programs\anaconda3\python.exe

1.2.2 我用 mac 時失敗了(可能是我用pyenv安裝的關係)

  • 我在 mac 上，即使照剛剛的操作，還是會失敗(尤其是我的 terminal shell常會吃不到)，那就需要打開 setting.json來設定。
  • 以下內容，整理至官方網站： https://code.visualstudio.com/docs/python/environments
  • 打開VS code後，輸入command+shift+p (⇧⌘P)，然後key入preferences: open user settings，開啟設定。然後照下面圖面做設定：
  • 打開json檔後，做以下的輸入：
{
    "jupyter.alwaysTrustNotebooks": true,
    "python.pythonPath": "/Users/hanklee/.pyenv/versions/3.8.0/bin/python",
    "terminal.integrated.env.osx": {
    "PATH": ""
    }
}
  • 這邊要注意的只有裡面的兩行一定要有：
    • "python.pythonPath": "/Users/hanklee/...."， 這邊的路徑，就是放你想要跑的版本。例如我都用pyenv來管理我的python version，那我可以在terminal用pyenv global 3.8.0來選定我要用3.8.0的版本後，再下which python，來得到這個版本的路徑。做完這行設定後，只要在VScode中點視窗右上角的run，他就會幫你下達這個指令：path/to/python/version path/to/this/file。
    • "PATH": ""，這個指令，是我google到的解法(連結)，主要是要解決，在VScode中，我開視窗下面的terminal，但執行python時，還是沒有執行到python.pythonPath的版本。
