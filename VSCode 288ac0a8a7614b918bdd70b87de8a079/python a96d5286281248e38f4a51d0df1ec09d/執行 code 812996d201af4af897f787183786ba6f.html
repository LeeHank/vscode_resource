執行 code

3.1 always先開個資料夾

  • 在 RStudio 的時候，我們很習慣開一個 project 來做事
  • 阿一個 project 就是一個資料夾的意思，所以對 VSCode也一樣，他希望你先開一個資料夾再來做事
  • 所以習慣上都會先開一個資料夾，然後新增一個 .py 檔 (就類比於 RStudio 先開個 project，再新增一個 .r 檔)，那就可以開始對這個 script 進行 coding了

3.2 逐行執行

  • 在 VSCode，都是用 shift + enter 來逐行執行 (左下角的齒輪/keyboard shortcuts，寫python runs，就會看到python的預設快捷鍵，是shift+enter，你可以改成rstudio的ctrl+enter，但建議不要啦)。他有兩種逐行執行的呈現方式：
    • 像 R console 一樣，他會幫你執行在最下面的 terminal頁面。這個的好處是乾淨(因為不會有新視窗)，但缺點是你如果是執行畫圖時，就很醜(他沒像RStudio那麼聰明，你在 R console 執行 plot()時，他自動幫你跳右邊的圖形呈現pannel)
    • 他會另開一個視窗，他叫 interactive window，其實就是啟動 jupyter notebook來執行你的這段code。他的好處是畫圖時就很方便(因為就像在Jupyter notebook裡畫圖)，但壞處就是多一個視窗很阿雜。

3.2.1 用 python terminal

  • 進到設定頁面：按 cmd+shift+p → user settings → jupyter send
  • 不要打勾，那 shift + enter 就會使用 python terminal
  • 開一個 .py 檔後，只要游標放在要執行的那一行，按 shift + enter ，就會在 console 開啟 Python 的 REPL (python terminal)，開始evaluate結果

3.2.2 用 jupyter interactive window

  • 進到設定頁面：按 cmd+shift+p → user settings → jupyter send
  • 把他打勾，那 shift + enter 就會用 jupyter interactive window。等等繼續做設定，就做出很類似RStudio的結果
  • 接著，我們要去設定 jupyter interactive window 的根目錄：
    • 一樣先進到設定裡面： cmd+shift+p → user settings → jupyter file root ，會看到下圖
      • 然後他原本的設定是 ${fileDirname} ，意思是，當我對 ~/my_project/hello_word.py 這個檔案逐行執行時，jupyter interactive window，會默認他目前所處的根目錄就是這個 file 的位子，也就是 ~/my_project/hello_word.py
      • 但我們希望他的根目錄，是在我的 project 的目錄，不是這個檔案的目錄。也就是，我希望跟目錄要在 ~/my_project 這裏。所以，我就把根目錄改成 ${workspaceFolder}
  • 開一個 .py 檔後，只要游標放在要執行的那一行，按 shift + enter ，就會開啟 jupyter interactive window，如下圖：
  • 點圖中的第3點，就會多出變數視窗(variable explorer)。
  • 再點data.frame的連結，就會跳出像R一樣的 View()
  • 如圖中，點完3後，就出現4的變數清單。點5這個dataframe，就跳出6的表格
